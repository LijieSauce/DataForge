## 1. 项目背景与目标

### 1.1 项目背景
在计算机视觉和深度学习领域，数据集的准备和管理是一项繁琐且耗时的工作。常见的痛点包括：
- 开源数据集格式多样（COCO、VOC、YOLO等），需要频繁转换
- 数据集质量参差不齐，需要人工清洗
- 缺少负样本，需要手动收集和生成
- 标注文件与图片需要配对管理
- 数据增强操作重复且机械

### 1.2 项目目标
构建一个**一站式图像数据集工具箱**，提供从数据采集、格式转换、质量清洗、数据增强到可视化预览的完整工作流，让用户通过图形界面完成所有操作，无需编写代码。

### 1.3 核心价值
- ✅ **零代码操作**：通过GUI界面完成所有任务
- ✅ **格式统一**：将各种格式转换为YOLO统一管理
- ✅ **质量保证**：人工清洗 + 可视化验证
- ✅ **模块独立**：每个功能独立运行，互不依赖

## 3. 模块详细设计

### 3.1 模块M1：网络图片收集

**设计目标**：快速获取大量图片作为数据集基础  

**功能描述**：
- 支持Bing图片搜索（通过Selenium模拟浏览器）
- 支持Pexels API（通过Token认证）
- 自动编号管理（0001.jpg, 0002.jpg...）
- 多线程并发下载，显示进度条

**关键流程**：
```
用户输入关键词 → 调用搜索API/打开搜索页 
→ 提取图片URL列表 → 多线程下载 
→ 按编号保存到输出目录 → 显示下载统计
```

**技术要点**：
- Selenium WebDriver（Bing）
- Requests + API Token（Pexels）
- ThreadPoolExecutor（多线程）
- 文件命名冲突检测

---

### 3.2 模块M2：格式转换

**设计目标**：统一数据集格式为YOLO，方便后续处理  

**功能描述**：
- **COCO → YOLO**：解析JSON标注，转换bbox格式
- **VOC → YOLO**：解析XML标注，转换bbox格式
- **YOLO → YOLO**：标准化 + 类别过滤
- **类别提取**：仅保留特定class_id的样本

**数据格式对比**：
| 格式 | 标注文件 | bbox表示 | 坐标系 |
|------|---------|---------|--------|
| COCO | JSON | [x, y, w, h] | 绝对像素 |
| VOC | XML | [xmin, ymin, xmax, ymax] | 绝对像素 |
| YOLO | TXT | [x_center, y_center, w, h] | 归一化比例 |

**转换公式**：
```
YOLO格式：
x_center = (bbox_x + bbox_w/2) / image_width
y_center = (bbox_y + bbox_h/2) / image_height
width = bbox_w / image_width
height = bbox_h / image_height
```

**关键流程**：
```
选择转换类型 → 读取源数据集标注 
→ 解析bbox和类别 → 类别过滤（可选）
→ 坐标归一化转换 → 复制图片+生成TXT标注 
→ 生成dataset.yaml配置文件
```

---

### 3.3 模块M3：数据增强

**设计目标**：将LabelMe等工具标注的JSON转为YOLO格式  

**功能描述**：
- 解析JSON标注（支持圆形circle、矩形rectangle）
- 计算标注区域的水平边界框（HBB）
- 转换为YOLO格式TXT文件
- 可选：抠出目标 + 合成到背景图（数据增强）

**关键流程**：
```
读取JSON标注 → 解析shapes字段 
→ 计算圆形/矩形的外接矩形 
→ 转换为YOLO归一化坐标 
→ 复制原图 + 生成TXT标注
```

---

### 3.4 模块M4：数据集划分

**设计目标**：将平铺的images/labels目录划分为train/val/test结构  

**功能描述**：
- 扫描平铺的images和labels目录
- 验证图片和标注文件配对
- 按比例随机划分（支持自定义比例）
- 移动文件到对应的train/val/test子目录

**目录结构转换**：
```
划分前：
dataset/
  ├── images/
  │   ├── 0001.jpg
  │   └── 0002.jpg
  └── labels/
      ├── 0001.txt
      └── 0002.txt

划分后：
dataset/
  ├── images/
  │   ├── train/
  │   ├── val/
  │   └── test/
  └── labels/
      ├── train/
      ├── val/
      └── test/
```

**关键流程**：
```
扫描images目录 → 查找对应labels文件 
→ 构建配对列表 → 按比例随机打乱 
→ 移动文件到train/val/test → 生成统计报告
```

---

### 3.5 模块M5：图像清洗

**设计目标**：人工筛选图片质量，删除模糊、错误的样本  

**功能描述**：
- 逐张显示图片 + YOLO标注框
- 提供三个操作按钮：保留 / 删除 / 待重标注
- 实时统计已处理数量
- 支持中途暂停和恢复

**用户交互**：
```
显示图片+标注框 → 用户判断质量 
→ 点击按钮（保留/删除/待重标注）
→ 自动跳转下一张 → 显示进度条
```

**技术要点**：
- OpenCV绘制标注框
- PyQt6嵌入图像显示控件
- 快捷键支持（A保留、D删除、S重标注）

---

### 3.6 模块M6：目标提取

**设计目标**：从图像中抠出标注区域，用于后续分析或合成  

**功能描述**：
- 读取JSON标注文件（LabelMe格式）
- 支持圆形和矩形标注
- 生成带透明通道的PNG图片（Alpha通道）
- 自动裁剪到最小包围矩形

**关键流程**：
```
读取JSON标注 → 解析shape字段 
→ 创建遮罩mask → 应用遮罩到原图 
→ 裁剪到bbox → 保存为PNG（带透明）
```

---

### 3.7 模块M7：负样本生成

**设计目标**：快速生成负样本（背景图片）补充数据集  

**功能描述**：
- 支持多个源文件夹
- 每个文件夹设置抽取比例
- 随机采样指定数量的图片
- 生成空白标注文件（.txt）

**关键流程**：
```
添加多个源目录 → 设置每个目录的比例 
→ 计算每个目录抽取数量 → 随机采样 
→ 复制图片到输出目录 → 生成空白TXT文件
```

**使用场景**：
- 目标检测需要负样本（不包含目标的背景图）
- 从多个数据源混合采样
- 快速扩充数据集规模

---

### 3.8 模块M8：可视化预览

**设计目标**：验证标注质量，快速定位标注错误  

**功能描述**：
- 从train/val/test中随机抽样
- 按class_id分组
- 在图片上绘制bbox + class文字
- 生成预览图保存到输出目录

**关键流程**：
```
扫描数据集 → 按类别收集标注文件 
→ 每类随机抽样N张 → 绘制bbox和文字 
→ 保存预览图 → 生成summary.txt统计
```

```
┌──────────────┐

└──────────────┘
       ↓
┌──────────────┐

└──────────────┘
       ↓
┌──────────────┐
│ 3.数据增强   │ → 补充标注（可选）
└──────────────┘
       ↓
┌──────────────┐
│ 4.图像清洗   │ → 人工筛选质量
└──────────────┘
       ↓
┌──────────────┐
│ 5.负样本生成 │ → 补充背景图（可选）
└──────────────┘
       ↓
┌──────────────┐
│ 6.数据集划分 │ → train/val/test
└──────────────┘
       ↓
┌──────────────┐
│ 7.可视化预览 │ → 验证标注质量
└──────────────┘
```

### 5.2 数据格式标准

**YOLO数据集标准结构**：
```
dataset/
├── images/
│   ├── train/
│   ├── val/
│   └── test/
├── labels/
│   ├── train/
│   ├── val/
│   └── test/
└── dataset.yaml
```

**dataset.yaml配置**：
```yaml
train: images/train
val: images/val
test: images/test

nc: 3  # 类别数量
names: ['class0', 'class1', 'class2']  # 类别名称
```

**TXT标注格式（每行一个目标）**：
```
<class_id> <x_center> <y_center> <width> <height>
```
*所有坐标均为归一化值（0-1之间）*

## 7. 扩展性设计

### 7.1 插件化架构
- 每个模块独立封装为Python类
- 通过统一接口调用：`execute(config) → result`
- 新增模块只需实现接口并注册到主界面

### 7.2 配置文件驱动
- 用户配置保存为JSON/YAML
- 支持预设模板（快速加载常用配置）
- 历史记录自动保存

### 7.3 批处理支持
- 支持命令行模式（无GUI运行）
- 支持批量任务脚本（YAML定义多步骤）

## 8. 风险与挑战

| 风险点 | 影响 | 应对方案 |
|--------|------|---------|
| Bing页面结构变化 | 无法抓取图片 | 提供手动URL输入备用 |
| 不同数据集目录结构差异 | 转换失败 | 智能探测+手动指定 |
| 大数据集处理慢 | 用户体验差 | 多线程+进度条反馈 |
| 标注格式不标准 | 解析错误 | 异常捕获+错误日志 |
